Data Structures / OO Project

Your boss at Interactive Geometrics, Inc wants you to make a proof-of-concept, prototype application for demonstrating how to find the area of various shapes. Using your knowledge of Java, of object-oriented programming, of data structures, and of either Processing or the Java2D graphics library, you set about making an application which allows the user to select 1) a particular kind of shape, and 2) a particular number of shapes. When the user selects the shape and number, shapes of random sizes get rendered to the screen, and the total area of the shapes gets output to the console. For the proof of concept, you decide to create a Pentagon shape, a Circle shape, and a Hexagon shape. Since other developers will be adding other shapes in the future, you decide to create interfaces for the core functionality which future developers will also be required to implement.

Here are the interfaces you will use:

public interface AreaInformation {
  public double calculateArea();
}

public interface RenderingAlgorithm {
  public void render();
}

The superclass of each of your geometric shapes will then implement these interfaces, requiring the subclasses to provide the calculateArea and render methods:

public abstract class GeometricShape implements AreaInformation, RenderingAlgorithm {
  double area;
  public double getArea() { return area; }
  public void setArea(double area) { this.area = area; }
}

public class Pentagon extends GeometricShape {
  //must provide a concrete method calculateArea and render to satisfy the contract of the interfaces
  public double calculateArea() {...}
  public void render() {...}

  //other stuff...
}
